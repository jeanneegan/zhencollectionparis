# Zhen Collection Philosophy

**Don't model categories. Model life.**

**不要建模分类，而要建模生命。**

因为我越来越觉得，巴黎臻藏真正想记录的，不是静态的资料，而是**艺术世界如何不断生长、不断连接、不断影响彼此。**

**我认为，这会成为巴黎臻藏区别于所有艺术数据库的核心。**

## Data Principles｜数据原则

### Principle 1｜每个成员都有两部分

Every member has:

- Profile｜自己的信息
- Relationships｜与世界的关系

### Principle 2｜平台记录关系，不只记录信息

巴黎臻藏不是普通资料库，而是艺术生态关系网络。

### Principle 3｜关系是核心资产

每一次代理、展览、收藏、策展、评论、采访、公众回应，都是一条关系记录。

### Principle 4｜内容也有自己的关系

作品、展览、对话、文章、出版物、视频，也都有自己的 Profile 和 Relationships。

## Core Architecture｜核心架构

巴黎臻藏不是固定职业列表，而是一个艺术生态关系网络。

### 1. Member｜成员

A Member is any person, organization, content, or entity that participates in the art ecosystem and can establish relationships with others.

成员可以是任何参与艺术生态、并能与其他成员建立关系的人、机构、内容或对象。

Member types are extensible.

成员类型是可扩展的。

Examples:

- Artist｜艺术家
- Gallery｜画廊
- Collector｜收藏家
- Curator｜策展人
- Critic｜评论家
- Museum｜博物馆
- Publisher｜出版社
- Media｜媒体
- Blogger / Content Creator｜博主 / 内容创作者
- Public｜公众
- Other｜其他

Every Member has:

- Profile｜自己的信息
- Relationships｜与世界的关系

---



### 2. Relationship / Event｜关系 / 事件

A Relationship records a real connection between Members.

关系记录成员之间真实发生的连接。

It should not be limited to a fixed list of roles.

它不应该被固定职业或固定关系类型限制。

Examples:

- Represented｜代理
- Exhibited｜展览
- Collected｜收藏
- Curated｜策展
- Reviewed｜评论
- Published｜出版
- Interviewed｜采访
- Collaborated｜合作
- Followed｜关注
- Visited Studio｜拜访工作室
- Responded｜回应
- Participated｜参与
- Recommended｜推荐
- Other｜其他

---



### 3. Content｜内容

Content is also a Member when it can have its own profile and relationships.

当内容本身可以拥有档案和关系时，它也可以成为 Member。

Examples:

- Artwork｜作品
- Exhibition｜展览
- Conversation｜对话
- Publication｜出版物
- Article｜文章
- Video｜视频
